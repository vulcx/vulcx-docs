# Error Codes

All error responses use the same format:

```json
{
  "success": false,
  "error": "error message"
}
```

## HTTP status codes

| Code | Meaning | When |
|------|---------|------|
| `200` | Success | Request processed successfully |
| `400` | Bad Request | Invalid parameters, missing required fields, insufficient liquidity |
| `401` | Unauthorized | No API key was provided |
| `403` | Forbidden | The key is invalid, disabled, or revoked — blocked by the key's origin allowlist — or a `quoteId` issued to a different key |
| `404` | Not Found | No route or pool found for the given parameters |
| `409` | Conflict | Firm-quote redemption failed: the pinned route is gone, or the price drifted past the firm margin |
| `410` | Gone | The `quoteId` expired (or is unknown) — request a fresh quote |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Transaction build failed, simulation error, or unexpected server issue |

## API errors

### Authentication errors (all `/api/v1/*` endpoints)

| Error message | HTTP | Cause |
|---------------|------|-------|
| `missing api key` | 401 | No `Authorization: Bearer` header or `?key=` parameter was sent |
| `invalid api key` | 403 | The key doesn't exist, is disabled, or was revoked |
| `origin required for this api key` | 403 | The key has an origin allowlist and the request sent no `Origin`/`Referer` |
| `origin not allowed for this api key` | 403 | The request's origin isn't on the key's allowlist |
| `rate limit exceeded` | 429 | Too many requests — see [Rate limits](/api-reference/rate-limits) |

### Quote errors (`GET /api/v1/quote`)

| Error message | HTTP | Cause |
|---------------|------|-------|
| `invalid inputMint` | 400 | `inputMint` is not a valid base58 public key |
| `invalid outputMint` | 400 | `outputMint` is not a valid base58 public key |
| `invalid amount` | 400 | Amount is not a valid positive number |
| `invalid swapMode` | 400 | `swapMode` must be `ExactIn` or `ExactOut` |
| `no route found` | 404 | No viable swap path exists |
| `no pool found` | 404 | No liquidity pool exists for one of the required hops |
| `insufficient liquidity` | 400 | Pools exist but don't have enough liquidity for the requested amount |

### Swap errors (`POST /api/v1/swap`)

| Error message | HTTP | Cause |
|---------------|------|-------|
| `invalid user wallet` | 400 | `userWallet` is not a valid base58 public key |
| All quote errors above | 400/404 | Same causes as quote endpoint |
| `simulation failed: ...` | 500 | Transaction simulation failed (details in message) |
| `failed to build transaction` | 500 | Internal error building the transaction |

### Instructions errors (`POST /api/v1/instructions`)

| Error message | HTTP | Cause |
|---------------|------|-------|
| `invalid user wallet` | 400 | `userWallet` is not a valid base58 public key |
| All quote errors above | 400/404 | Same causes as quote endpoint |
| `failed to build instructions` | 500 | Internal error building the instructions |
| `missing pool data` | 500 | Pool account data unavailable (temporary) |

### Firm-quote errors (`POST /api/v1/swap`, `POST /api/v1/instructions` with `quoteId`)

| Error message | HTTP | Cause | Recovery |
|---------------|------|-------|----------|
| `quote expired: request a fresh quote` | 410 | The `quoteId` is past `validForMs` (or past `firmForMs` when `firm: true`), was evicted, or never existed | Fetch a fresh quote and retry with the new `quoteId` |
| `quoted route is no longer available: request a fresh quote` | 409 | The pinned route's pool disappeared — or, with `firm: true`, the price drifted past the firm margin | Fetch a fresh quote; with `firm`, retry immediately (the window is short) |
| `quoteId was issued to a different api key` | 403 | The quote was minted under another API key | Re-quote with your own key |
| `request does not match the quoted pair, amount, or swap mode` | 400 | The swap request's pair/amount/swapMode differ from the original quote | Send the exact quoted request, or drop `quoteId` |
| `firm redemption requires a quoteId` | 400 | `firm: true` without a `quoteId` | Include the `quoteId` from the quote you're redeeming |

See [Firm quotes](/docs/swap/firm-quotes) for how `quoteId` and `firm` work.

## Simulation result errors

When `skipSimulation` is `false`, the swap response includes a `simulation` object:

| Field | Type | Meaning |
|-------|------|---------|
| `success` | boolean | Whether simulation passed |
| `error` | string | Error message if simulation failed |
| `insufficientFunds` | boolean | Wallet doesn't have enough balance |
| `slippageExceeded` | boolean | Price moved beyond tolerance |
| `computeUnitsConsumed` | integer | Compute units (CU) used during simulation |
| `logs` | string[] | Program log output |

## On-chain errors

These errors occur after submitting a signed transaction to the network:

| Error | Cause | Recovery |
|-------|-------|----------|
| `SlippageExceeded` | Price changed between build and execution | Get a new quote and rebuild |
| `InsufficientFunds` | Balance changed after simulation | Verify balance, then retry |
| `BlockhashExpired` | Transaction wasn't confirmed in time | Rebuild with fresh blockhash |
| `InvalidAccountData` | Pool state changed | Get a new quote (pool may have been migrated) |
